## 1. Goal
Implement serialized `git worktree` initialization safely and execute SQL/Environment isolation to ensure concurrent overlapping multi-spec agents natively avoid locking bugs without halting the orchestrator event loop.

## 2. Code Changes

### [MODIFY] `src/specweaver/core/flow/_base.py`
**Goal**: Expand `RunContext` natively so each isolated sandboxed PipelineRun controls its mocked network/DB footprint invisibly. Add context tracking for Git branching logic to support human readability without overlapping branches.

- **Modifications**:
  - Add `env_vars: dict[str, str] | None = Field(default_factory=dict)` to `RunContext`. This maintains pristine isolation bounded specifically per sub-pipeline payload.
  - Modify `RunContext` to accept `context.pipeline_name` string so that `runner.py` can fetch it directly for isolated deterministic string construction.

### [MODIFY] `src/specweaver/core/flow/state.py`
**Goal**: Integrate the declarative resource reservation state for the SQLite gating framework.

- **Modifications**:
  - Extend the `GateType` enum to explicitly include `RESERVE` alongside `AUTO`, `HITL`, and `JOIN`.
  - Ensure the evaluation pipeline routes `RESERVE` properly.

### [NEW] `src/specweaver/core/flow/reservation.py`
**Goal**: Establish a thread-safe atomic lock schema in SQLite via Pydantic/SQLAlchemy boundaries.

- **Modifications**:
  - Define a new DB service `SQLiteReservationSystem`.
  - Create table definition: `CREATE TABLE IF NOT EXISTS sw_reservations (resource_id TEXT PRIMARY KEY, run_id TEXT NOT NULL, expires_at DATETIME)` guaranteeing atomic uniqueness exclusivity.
  - Implement `.acquire(resource_id, run_id)` leveraging raw `try: INSERT ... except sqlite3.IntegrityError: return False`. It MUST contain extensive logging reporting `Race condition lost naturally by <run_id> for lock <resource_id>`.
  - Implement `.release(resource_id, run_id)` which deletes the lock cleanly.

### [MODIFY] `src/specweaver/core/flow/runner.py`
**Goal**: Thread the Sandboxing components together.

- **Modifications**:
  - **Environment Intercepts**: When LLM calls or shell executables spawn under `_execute_loop`, inject `.env_vars` generated by the parent runtime safely.
  - **Git Isolation Prefixing**: Change worktree creation string generation. Refactor:
    `branch = f"sf-{clean_pipeline}-{task_id}"` utilizing `.replace(' ', '_')` sanitization.
  - **Database Cleanup Hooks**: Implement an explicit `finally:` hook block executing the DB `.release` command. This ensures a catastrophic agent failure in a sub-shell does not leave a Zombie resource lock forever.

### [MODIFY] `src/specweaver/core/flow/gates.py`
**Goal**: Interpret `GateType.RESERVE` behavior.

- **Modifications**:
  - Implement `evaluate_reserve()`. If `SQLiteReservationSystem.acquire(...)` fails natively (due to atomic Unique lock block), the Gate Evaluator emits a meaningful log exactly defining: `[run_id=X] Sandbox environment reservation for {resource_id} is locked by another agent. Yielding verdict=PARK`.
  - Return `verdict="park"` gracefully back into `_execute_loop`. The Parent Topological DAG wave dispatcher automatically polls and `resumes` parked operations iteratively.

## 3. Developer / HITL Architecture Approvals
- **Git Worktree Naming Space**: HITL explicitly approved `Pipeline + Task ID` (Option C) over static components and UUIDs, ensuring zero lock collisions and high human debug readout visibility.
- **SQLite Reservation Strategy**: HITL explicitly approved using a native `GateType.RESERVE` pipeline architecture (Option C) bounding an atomic SQLite database unique constraint (`IntegrityError`). It completely abandons arbitrary looping sleep constraints inside Python, protecting system performance safely. Logging was strictly emphasized for collision fallbacks.
- **SW_PORT_OFFSET Variables**: Handled strictly via explicit `RunContext.env_vars` maps locally inside the PipelineRunner rather than utilizing global overrides (Option A).

## 4. Backlog / Deferred
- Rate throttling (`asyncio.Semaphore()`) against LLM APIs will be built explicitly in SF-3 (Parallel Engine Hardening).
- Global Documentation Synthesis triggers (`GateType.JOIN`) will also be enforced in SF-3.

> Ensure all DB calls for `sw_reservations` aggressively wrap `sqlite3` driver errors specifically to avoid breaking the core flow event loop when an overlapping test starts.
